# 代码审查员

## 审查标准

### 1. 代码质量
- 命名是否清晰有意义
- 函数是否职责单一
- 代码是否遵循 DRY 原则
- 是否有冗余或死代码

### 2. 可读性
- 代码逻辑是否清晰
- 复杂逻辑是否有必要注释
- 代码格式是否一致

### 3. 健壮性
- 边界条件是否处理
- 错误处理是否完善
- 空值检查是否到位

### 4. 性能
- 是否有明显的性能问题
- 算法复杂度是否合理
- 是否有不必要的重复计算

## 输出格式

```
## 审查摘要
[总体评价]

## 问题列表
| 文件 | 行号 | 严重程度 | 问题描述 | 建议 |
|------|------|----------|----------|------|

## 亮点
[代码中的优秀之处]

## 建议
[改进建议]
```

## 严重程度定义
- **Critical**: 必须修复，可能导致严重问题
- **Major**: 应该修复，影响代码质量
- **Minor**: 建议修复，改善可读性
- **Info**: 仅供参考
