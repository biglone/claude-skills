# 简历撰写助手

## 简历原则

### 核心原则
```
1. 量化成果 - 用数字说话
2. 关键词匹配 - 针对职位定制
3. 简洁清晰 - 一页为佳
4. 突出价值 - 你能带来什么
```

### STAR 原则写经历
```
S - Situation（情境）：背景是什么
T - Task（任务）：要完成什么
A - Action（行动）：采取了什么行动
R - Result（结果）：取得了什么成果

示例：
"在用户增长放缓的背景下（S），
负责设计新的用户获取策略（T），
主导了3个渠道的A/B测试和优化（A），
3个月内将新用户注册量提升40%（R）。"
```

## 优化技巧

### 动作词列表
```
领导类：主导、带领、统筹、推动、发起
执行类：实现、完成、交付、落地、执行
创新类：设计、开发、创建、优化、改进
分析类：分析、评估、研究、调研、诊断
沟通类：协调、沟通、对接、推进、促成
```

### 量化成果
```
✗ "提升了系统性能"
✓ "将系统响应时间从500ms优化至100ms，性能提升80%"

✗ "负责用户增长"
✓ "3个月内将DAU从10万提升至25万，增长150%"

✗ "管理团队"
✓ "管理8人团队，完成12个项目交付，团队满意度95%"

可量化的维度：
- 时间：节省了X小时/天/周
- 金额：节省/创造了X元
- 比例：提升/降低了X%
- 数量：服务了X用户/处理了X订单
- 排名：团队/公司/行业第X
```

### 关键词优化
```
1. 分析目标职位描述
2. 提取关键技能和要求
3. 在简历中自然融入关键词
4. 确保ATS（简历筛选系统）友好

示例：
职位要求："熟悉敏捷开发、有项目管理经验"
简历："使用Scrum敏捷方法管理3个并行项目"
```

### 格式建议
```
1. 长度：应届生1页，有经验者1-2页
2. 字体：清晰易读（宋体、黑体、微软雅黑）
3. 字号：姓名14-16pt，正文10-12pt
4. 间距：留白适当，不要过于拥挤
5. 格式：PDF格式发送，避免格式错乱
```

## 常见问题

### 职业空白期
```
解决方案：
1. 诚实但积极框架
   "2020-2021：全职照顾家人期间，通过在线课程学习了[技能]"

2. 强调这段时间的收获
   - 学习了什么
   - 做了什么有价值的事
```

### 频繁跳槽
```
解决方案：
1. 突出每段经历的成长和价值
2. 在求职信中解释职业规划
3. 强调寻找长期发展的诚意
```

### 转行/转岗
```
解决方案：
1. 突出可迁移技能
2. 强调与新岗位的连接点
3. 展示为转行所做的准备
4. 用项目/副业证明能力
```

## 求职信模板

```markdown
尊敬的招聘负责人：

您好！我是[姓名]，对贵公司[职位名称]一职非常感兴趣。

**为什么是这个职位**
[表达对职位/公司的了解和热情，1-2句]

**为什么是我**
我有X年[领域]经验，核心优势是：
1. [优势1]：[具体证据]
2. [优势2]：[具体证据]

**我能带来什么**
基于我在[领域]的经验，我相信能够：
- [贡献1]
- [贡献2]

期待有机会进一步沟通，详细展示我的能力和热情。

感谢您的时间！

[姓名]
[联系方式]
```

## 输出格式

优化简历时，提供：
1. **结构建议** - 简历架构优化
2. **内容优化** - 具体措辞改进
3. **量化建议** - 如何量化成果
4. **匹配分析** - 与目标职位的匹配度
5. **完整简历** - 优化后的版本
