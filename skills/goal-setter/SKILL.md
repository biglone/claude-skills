# 目标设定助手

## 目标设定方法

### SMART 原则
```
S - Specific（具体的）
    ❌ "我要学编程"
    ✅ "我要学会用 Python 进行数据分析"

M - Measurable（可衡量的）
    ❌ "提高英语水平"
    ✅ "雅思成绩达到 7 分"

A - Achievable（可实现的）
    考虑：资源、能力、时间是否足够

R - Relevant（相关的）
    与更大的目标或价值观一致

T - Time-bound（有时限的）
    ❌ "有空的时候减肥"
    ✅ "3个月内减重10斤"
```

### OKR 框架
```
O - Objective（目标）
    定性描述，鼓舞人心

KR - Key Results（关键结果）
    定量指标，可衡量

示例：
O: 成为优秀的全栈开发者
├─ KR1: 完成 3 个全栈项目并部署上线
├─ KR2: 前端框架熟练度达到可独立开发复杂页面
└─ KR3: 通过 AWS 云认证考试
```

## 目标拆解方法

### 从大到小拆解
```
年度目标
    ↓
季度目标（OKR）
    ↓
月度重点
    ↓
周目标
    ↓
每日任务
```

### 倒推法
```
最终目标：12月31日达成 [X]
    ↑
Q4需要：[完成什么]
    ↑
Q3需要：[完成什么]
    ↑
Q2需要：[完成什么]
    ↑
Q1需要：[完成什么]
    ↑
现在开始：[第一步行动]
```

### 障碍预判
```
目标：[描述]

可能的障碍：
1. [障碍1]
   - 预防措施：
   - 如果发生：

2. [障碍2]
   - 预防措施：
   - 如果发生：

支持资源：
- [资源1]
- [资源2]
```

## 目标追踪

### 进度可视化
```
目标进度：[████████░░░░░░░░░░░░] 40%

月度进度：
1月 ████████████ 100%
2月 ████████░░░░  67%
3月 ████░░░░░░░░  33%
4月 ░░░░░░░░░░░░   0%
```

### 定期检查问题
```
每周检查：
- 本周为这个目标做了什么？
- 进度是否符合预期？
- 下周需要做什么？

每月检查：
- 这个目标还重要吗？
- 需要调整吗？
- 资源是否足够？

每季检查：
- 目标完成度如何？
- 学到了什么？
- 下季度如何调整？
```

## 输出格式

设定目标时，提供：
1. **清晰的目标陈述** - SMART 格式
2. **可衡量的指标** - 具体数值
3. **分解的子目标** - 可执行的步骤
4. **时间节点** - 明确的截止日期
5. **追踪机制** - 如何监控进度
