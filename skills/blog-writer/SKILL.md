# 博客写作助手

## 博客类型

### 1. 技术博客
- 技术教程
- 问题解决方案
- 技术趋势分析
- 开源项目介绍

### 2. 产品博客
- 产品发布
- 功能更新
- 使用案例
- 行业洞察

### 3. 个人博客
- 经验分享
- 学习笔记
- 观点评论
- 生活感悟

## 文章结构

### 经典结构
```
1. 引言 (Hook) - 抓住读者注意力
2. 问题 (Problem) - 阐述痛点
3. 解决方案 (Solution) - 提供答案
4. 详细说明 (Details) - 展开论述
5. 总结 (Conclusion) - 回顾要点
6. 行动号召 (CTA) - 引导下一步
```

### 教程结构
```
1. 介绍 - 为什么学习这个
2. 前提条件 - 需要准备什么
3. 步骤 1-N - 具体操作
4. 常见问题 - 可能遇到的坑
5. 总结 - 学到了什么
6. 延伸阅读 - 进一步学习
```

### 观点文章结构
```
1. 引入话题 - 背景介绍
2. 提出观点 - 核心论点
3. 论据 1 - 支持观点
4. 论据 2 - 进一步支持
5. 反驳 - 回应可能的质疑
6. 结论 - 总结观点
```

## 写作技巧

### 标题写作
```markdown
# 好标题特点
- 明确：告诉读者会得到什么
- 吸引：激发好奇心
- 具体：包含具体数字或关键词

# 标题模板
- "如何 [达成某目标]"
- "[数字] 个 [技巧/方法/工具] 帮你 [达成目标]"
- "[问题]？这里有 [解决方案]"
- "为什么 [某现象]（以及如何 [应对]）"
- "[初学者/高级] 指南：[主题]"

# 示例
✓ "5 个技巧让你的 React 应用性能提升 50%"
✓ "为什么我从 Webpack 迁移到 Vite（以及你是否应该）"
✗ "关于 React 的一些想法"（太模糊）
```

### 开头写作
```markdown
# 好的开头方式

1. 提问式
"你是否遇到过这样的情况：项目上线后，用户抱怨加载太慢？"

2. 数据式
"根据统计，53% 的移动用户会在页面加载超过 3 秒时离开。"

3. 故事式
"上周，我们的服务器突然宕机，而我正在度假..."

4. 痛点式
"处理异步代码一直是 JavaScript 开发者的噩梦。"

5. 承诺式
"读完这篇文章，你将学会用 Docker 部署任何应用。"
```

### 段落写作
```markdown
# 原则
- 一个段落一个主题
- 首句点明主题
- 3-5 句为宜
- 段落间有逻辑过渡

# 过渡词
- 递进：此外、更重要的是、不仅如此
- 转折：然而、但是、不过
- 因果：因此、所以、由于
- 举例：例如、比如、具体来说
- 总结：总之、综上所述、简而言之
```

### 结尾写作
```markdown
# 好的结尾方式

1. 总结要点
"回顾一下，我们学习了：1) ... 2) ... 3) ..."

2. 行动号召
"现在就试试这个方法，在评论区告诉我你的结果！"

3. 思考问题
"你会选择哪种方案？为什么？"

4. 展望未来
"随着 AI 的发展，这个领域将会..."

5. 资源链接
"想深入学习？这里有一些推荐资源..."
```

## SEO 优化

### 关键词
- 在标题中包含关键词
- 在前 100 字中出现关键词
- 合理使用同义词和相关词
- 避免关键词堆砌

### 元数据
```markdown
---
title: "文章标题 | 网站名"
description: "150 字以内的摘要"
keywords: "关键词1, 关键词2"
---
```

### 可读性
- 使用小标题（H2, H3）
- 段落简短
- 使用列表和表格
- 添加图片和代码块

## 输出格式

根据用户需求的博客类型，输出包含：
1. 吸引人的标题
2. 清晰的结构
3. 有价值的内容
4. 合适的长度（通常 1000-2500 字）
5. 行动号召
