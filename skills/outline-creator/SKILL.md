# 大纲创建助手

将零散想法整理成逻辑清晰、层级稳定、可直接写作的结构。

## 核心流程

1. 定义目标与受众
2. 选择组织逻辑
3. 生成层级骨架
4. 填充关键要点
5. 做一致性检查

## 1) 定义目标与受众

- 目标：解释、说服、教学、决策
- 受众：初学者、专业读者、管理层
- 产出：文章、报告、演示、方案

## 2) 选择组织逻辑

优先选择一条主逻辑：

- 时间顺序
- 问题-方案
- 结论先行
- 对比分析
- 流程步骤

不要在同一层混用多个主逻辑。

## 3) 生成层级骨架

- 一级：3-7 个模块
- 二级：每个一级下 2-5 个要点
- 三级：用于支撑证据或示例

保持同层并列关系一致，避免交叉重叠。

## 4) 填充关键要点

每个节点补足：

- 结论句（这个节点要表达什么）
- 证据点（数据/案例/定义）
- 转场句（与上下文连接）

## 5) 一致性检查

- 是否覆盖目标问题
- 是否存在重复节点
- 是否有“标题正确但内容空洞”的节点
- 是否能直接用于写作/演讲

## 输出格式

1. 结构图（1-3 级）
2. 每节点核心结论
3. 关键证据与示例建议
4. 结构风险与修订建议

- 需要更细的层级拆分示例
