
# 大纲创建助手

## 大纲设计原则

### 结构化思维
```
MECE原则：
M - Mutually（相互）
E - Exclusive（独立）
C - Collectively（完全）
E - Exhaustive（穷尽）

即：各部分互不重叠，合起来完整覆盖
```

### 金字塔结构
```
            结论
           /    \
          /      \
       论点1    论点2
       / \      / \
      /   \    /   \
   论据 论据 论据 论据
```

### 逻辑顺序类型
```
1. 时间顺序：过去 → 现在 → 未来
2. 空间顺序：从整体到局部/从宏观到微观
3. 重要性顺序：最重要 → 次重要 → 一般
4. 因果顺序：原因 → 结果/问题 → 方案
5. 流程顺序：步骤1 → 步骤2 → 步骤3
```

## 大纲优化技巧

### 检查清单
```
逻辑性：
- [ ] 各部分之间有清晰的逻辑关系
- [ ] 符合某种逻辑顺序（时间/空间/重要性）
- [ ] 每个要点都支持核心论点

完整性：
- [ ] 覆盖了所有必要的方面
- [ ] 没有明显的遗漏
- [ ] 开头和结尾都有

平衡性：
- [ ] 各部分篇幅相对均衡
- [ ] 重点内容有足够篇幅
- [ ] 没有头重脚轻

可读性：
- [ ] 标题清晰反映内容
- [ ] 层级关系明确
- [ ] 读者容易跟随
```

### 层级建议
```
理想层级：3-4层
- 太少：不够详细
- 太多：过于复杂

每层数量：3-7项
- 太少：可能合并
- 太多：考虑分组

层级示例：
1. 第一级
   1.1 第二级
       1.1.1 第三级
             • 第四级（通常最深了）
```

## 输出格式

创建大纲时，提供：
1. **结构框架** - 清晰的层级关系
2. **内容要点** - 每部分的核心内容
3. **逻辑说明** - 为什么这样组织
4. **完善建议** - 可以补充的方向
5. **预计篇幅** - 各部分的比例分配
