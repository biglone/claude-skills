# 演示文稿助手

## 演示设计原则

### 10-20-30 法则
```
10 - 不超过10页幻灯片
20 - 演讲不超过20分钟
30 - 字体不小于30号
```

### 金字塔原则
```
        结论
       /    \
      /      \
   论点1    论点2
   / \      / \
 证据 证据  证据 证据
```

### 故事弧线
```
引入 → 冲突 → 转折 → 高潮 → 解决 → 行动
```

## 设计技巧

### 每页结构
```
标题：告诉观众这页讲什么
    ↓
核心信息：1-3个关键点
    ↓
支撑材料：数据/图表/例子
    ↓
过渡：连接到下一页
```

### 文字处理
```
原则：
- 每页文字 ≤ 6行
- 每行文字 ≤ 6个词
- 用关键词代替完整句子

✗ "我们需要确保在第三季度之前完成所有用户调研工作"
✓ "Q3前完成用户调研"
```

### 数据可视化
```
选择合适的图表：
- 比较：柱状图
- 趋势：折线图
- 占比：饼图
- 关系：散点图
- 流程：流程图

原则：
- 一图一观点
- 标注关键数据
- 使用对比色突出重点
```

### 视觉一致性
```
保持统一：
- 字体：标题1种，正文1种
- 颜色：主色+辅色+点缀色（共3-4色）
- 间距：统一的页边距和行间距
- 图标：风格统一的图标集
```

## 演讲技巧

### 开场
```
1. 提问法："你们有没有遇到过...?"
2. 故事法："去年，我们的一个客户..."
3. 数据法："90%的公司都面临这个问题"
4. 对比法："想象一下，如果..."
```

### 过渡
```
承上：
"刚才我们讨论了[上一部分]"

启下：
"接下来让我们看看[下一部分]"
"那么，如何解决这个问题呢？"
```

### 结尾
```
1. 总结法：回顾3个核心观点
2. 行动法：明确告诉观众下一步
3. 回扣法：呼应开头的问题或故事
4. 愿景法：描绘美好未来
```

## 输出格式

设计演示文稿时，提供：
1. **完整大纲** - 每页的标题和要点
2. **内容建议** - 每页的详细内容
3. **设计提示** - 可视化和排版建议
4. **演讲备注** - 每页的讲述要点
