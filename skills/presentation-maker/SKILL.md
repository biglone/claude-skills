# 演示文稿助手

将模糊主题快速转成可讲、可看、可执行的演示结构。

## 核心流程

1. 明确场景与目标
2. 定义受众与关键决策
3. 搭建叙事主线
4. 规划逐页结构
5. 生成讲稿提示与行动项

## 1) 明确场景与目标

至少确认以下信息：

- 场景：汇报、路演、培训、复盘、提案
- 时长：5/15/30 分钟或其他
- 目标：告知、说服、决策、行动
- 输出形式：仅大纲或逐页内容

若信息不足，先给默认假设并标注待确认项。

## 2) 定义受众与关键决策

- 受众层级：一线执行、管理层、跨部门、客户
- 关注点：收益、成本、风险、进度、资源
- 关键问题：听众听完后要做什么决定

每页内容都要服务同一个决策目标。

## 3) 搭建叙事主线

默认结构：

- 问题与背景
- 关键洞察
- 方案与取舍
- 计划与资源
- 风险与应对
- 结论与行动

避免堆砌信息。每页只保留一个主信息点。

## 4) 规划逐页结构

每页输出统一格式：

- `页面标题`
- `核心结论`
- `支撑证据`（数据/案例/图示）
- `视觉建议`（图表、流程图、对比表）
- `演讲备注`（30-60 秒可讲述要点）

## 5) 生成讲稿提示与行动项

- 首尾页给出完整话术骨架
- 中间页给出转场句
- 结尾页给出明确行动建议（负责人 + 时间）

## 质量检查清单

- 是否能在 30 秒说明整场结论
- 是否每页仅 1 个主结论
- 是否存在无结论的数据页
- 是否有可执行下一步

## 输出格式

1. 受众与目标假设
2. 演示主线
3. 逐页大纲（标题/要点/视觉建议/演讲备注）
4. 风险与备选说法
5. 结尾行动项

## 渐进加载（按需读取）

仅在命中对应需求时读取：`references/full-guide.md`

- 需要完整方法论（10-20-30、金字塔、故事弧线）
- 需要行业/场景模板（工作汇报、项目提案、路演等）
- 需要高级演讲技巧与开场结尾话术库
