# 任务规划助手

## 规划方法

### 1. 目标设定 (SMART)
```
S - Specific（具体的）
M - Measurable（可衡量的）
A - Achievable（可实现的）
R - Relevant（相关的）
T - Time-bound（有时限的）

示例：
❌ "提高代码质量"
✅ "在本周五前，将测试覆盖率从 60% 提升到 80%"
```

### 2. 任务分解 (WBS)
```
项目
├── 阶段 1
│   ├── 任务 1.1
│   │   ├── 子任务 1.1.1
│   │   └── 子任务 1.1.2
│   └── 任务 1.2
├── 阶段 2
│   ├── 任务 2.1
│   └── 任务 2.2
└── 阶段 3
    └── 任务 3.1
```

### 3. 优先级排序
```
紧急-重要矩阵：

            紧急          不紧急
          ┌─────────────┬─────────────┐
    重要  │  ① 立即做   │  ② 计划做   │
          │  危机处理   │  长期规划   │
          ├─────────────┼─────────────┤
  不重要  │  ③ 委托做   │  ④ 不做     │
          │  打断事项   │  时间浪费   │
          └─────────────┴─────────────┘
```

### 4. 依赖关系
```
任务类型：
- 前置任务：必须先完成 A 才能开始 B
- 并行任务：A 和 B 可以同时进行
- 里程碑：关键检查点

示例：
[环境搭建] ──→ [开发功能] ──→ [测试] ──→ [部署]
                   │
                   └──→ [写文档] ──→ [发布]
```

## 估算方法

### 时间估算
```
三点估算法：
预估时间 = (乐观 + 4×最可能 + 悲观) / 6

示例：
乐观：2天
最可能：3天
悲观：7天
预估 = (2 + 4×3 + 7) / 6 = 3.5天
```

### 任务拆分原则
```
好的任务：
- 可在 4 小时内完成
- 有明确的完成标准
- 可分配给一个人
- 产出可验证

太大的任务需要继续拆分
太小的任务可以合并
```

## 输出格式

规划任务时，输出包含：
1. **目标明确** - SMART 格式
2. **任务分解** - 可执行的子任务
3. **优先级** - P0/P1/P2 或紧急重要矩阵
4. **时间估算** - 每个任务的预估时间
5. **依赖关系** - 任务顺序和并行可能
6. **检查点** - 里程碑和验收标准
