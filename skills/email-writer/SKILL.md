# 邮件撰写助手

## 邮件写作原则

### 5C 原则
```
Clear    - 清晰：一个邮件一个主题
Concise  - 简洁：尽量控制在5句话内
Correct  - 正确：信息准确，无错别字
Courteous - 礼貌：尊重对方
Complete  - 完整：包含所有必要信息
```

### 结构框架
```
1. 主题行 - 概括核心内容
2. 称呼 - 恰当的问候
3. 开头 - 说明目的（1-2句）
4. 正文 - 具体内容
5. 行动项 - 需要对方做什么
6. 结尾 - 礼貌收尾
7. 签名 - 身份信息
```

## 邮件模板

### 工作汇报邮件
```markdown
主题：[项目/工作] 周/月进度汇报 - [日期]

[称呼]：

现将[时间段]工作进展汇报如下：

## 本期完成

1. **[任务1]**：[简要描述成果]
2. **[任务2]**：[简要描述成果]

## 进行中

1. **[任务]**：预计[日期]完成

## 需要支持

- [需要协助的事项]

## 下期计划

- [计划事项]

如有问题，请随时联系。

[签名]
```

### 请求审批邮件
```markdown
主题：请批准：[具体事项]

[称呼]：

申请[具体事项]，详情如下：

**事项**：[描述]
**原因**：[为什么需要]
**时间**：[需要的时间]
**预算**：[如需要]
**影响**：[对工作的影响]

附件包含相关材料，请审批。

[签名]
```

### 会议邀请邮件
```markdown
主题：会议邀请：[会议主题] - [日期时间]

[称呼]：

诚邀您参加以下会议：

**会议主题**：[主题]
**时间**：YYYY年MM月DD日 HH:MM - HH:MM
**地点**：[地点/会议链接]
**参会人**：[人员列表]

**会议议程**：
1. [议题1]（X分钟）
2. [议题2]（X分钟）
3. [议题3]（X分钟）

**会前准备**：
- [需要准备的材料]

请确认是否能够参加。如有时间冲突，请告知。

[签名]
```

### 跟进催办邮件
```markdown
主题：友情提醒：[事项] - 请于[日期]前反馈

[称呼]：

关于[之前邮件/事项的简要描述]，想跟您确认一下进展。

**原始请求**：[简要说明]
**截止日期**：[日期]
**当前状态**：待您[具体行动]

如有任何问题，请告知。如已处理，请忽略此邮件。

感谢您的支持！

[签名]
```

### 道歉邮件
```markdown
主题：关于[事项]的致歉

[称呼]：

就[具体问题]向您致歉。

**发生了什么**：[简要说明问题]
**影响**：[对您造成的影响]
**原因**：[问题的原因]

**补救措施**：
1. [已采取的措施]
2. [后续计划]

**预防措施**：
[如何避免再次发生]

再次为此造成的不便深表歉意。

[签名]
```

### 感谢邮件
```markdown
主题：感谢：[具体事项]

[称呼]：

衷心感谢您[具体帮助/贡献]。

[详细说明感谢的原因和对您的帮助]

[表达对未来合作的期望]

再次感谢！

[签名]
```

### 拒绝邮件（委婉）
```markdown
主题：关于[请求事项]的回复

[称呼]：

感谢您[请求的内容]。

经过仔细考虑，[说明情况]，目前无法[满足请求]。

**原因**：[简要说明，如适用]

**替代方案**：
- [可能的替代建议]
- [其他选项]

希望这个回复不会给您带来太多不便。如有其他需要，请随时联系。

[签名]
```

### 介绍/推荐邮件
```markdown
主题：介绍：[人名] - [领域/关系]

[被介绍双方]：

我来介绍一下你们认识：

**[人名A]**：[职位/背景]，[相关专长或成就]

**[人名B]**：[职位/背景]，[相关专长或成就]

你们在[共同话题/合作机会]方面应该有很多可以交流的。

我已将双方的联系方式附上，接下来请你们直接联系。

祝交流愉快！

[签名]
```

## 邮件优化技巧

### 主题行写作
```
✓ 好的主题行：
- "请批准：Q2营销预算 - 需12月5日前确认"
- "会议纪要：产品评审会 - 附行动项"
- "紧急：服务器问题已修复 - 请验证"

✗ 差的主题行：
- "请看一下"
- "问题"
- "关于昨天的事"
```

### 正文优化
```
1. 开门见山
   ✗ "希望这封邮件找到你一切都好..."
   ✓ "关于[事项]，需要您的确认。"

2. 使用列表
   ✗ "首先...其次...然后...另外...最后..."
   ✓ 1. 2. 3. 或 • • •

3. 突出重点
   - 使用加粗标记关键信息
   - 截止日期、金额等用明确格式

4. 明确行动
   ✗ "请您考虑一下"
   ✓ "请于周五前回复确认"
```

### 语气调整
```
正式 ←──────────────────────→ 随意

"请您审阅"  "麻烦看下"  "帮忙看看"  "看一眼"

根据收件人选择：
- 上级/客户：较正式
- 平级同事：适度正式
- 熟悉伙伴：可随意
```

## 特殊场景

### 邮件回复
```
1. 及时回复（24小时内）
2. 引用原文关键部分
3. 逐点回应问题
4. 如需时间处理："收到，预计[时间]回复详情"
```

### 群发邮件
```
- TO：需要行动的人
- CC：需要知晓的人
- BCC：隐藏收件人（谨慎使用）

避免：
- 全体回复无关紧要的内容
- 抄送过多人
```

### 英文邮件常用语
```
开头：
- I hope this email finds you well.
- Following up on our conversation...
- I'm writing to inquire about...

正文：
- Please find attached...
- I would appreciate if you could...
- Could you please confirm...

结尾：
- Please let me know if you have any questions.
- Looking forward to hearing from you.
- Thank you for your time and consideration.
```

## 输出格式

写邮件时，提供：
1. **主题行** - 清晰概括的标题
2. **正文内容** - 结构化的邮件正文
3. **优化建议** - 改进措辞的提示
4. **注意事项** - 发送前的检查点
