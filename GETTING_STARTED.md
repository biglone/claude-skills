## 安装

### 方式 1：一键安装（推荐）

**macOS / Linux:**
```bash
curl -fsSL https://raw.githubusercontent.com/biglone/agent-skills/main/scripts/install.sh | bash
```

**Windows (PowerShell):**
```powershell
irm https://raw.githubusercontent.com/biglone/agent-skills/main/scripts/install.ps1 | iex
```

**Windows (cmd):**
```cmd
powershell -NoProfile -ExecutionPolicy Bypass -Command "Invoke-RestMethod https://raw.githubusercontent.com/biglone/agent-skills/main/scripts/install.ps1 | Invoke-Expression"
```

说明：`irm` 是 PowerShell 命令别名，在 `cmd` 里不能直接执行。

### 方式 2：手动安装

```bash
git clone https://github.com/biglone/agent-skills.git
cd agent-skills

# Claude Code
cp -r skills/* ~/.claude/skills/
cp -r workflows/* ~/.claude/workflows/

# Codex CLI
cp -r skills/* ~/.codex/skills/
cp -r workflows/* ~/.codex/workflows/
```

### 验证安装

```bash
# 在 Claude Code 中
ls ~/.claude/skills/requirements-doc
ls ~/.claude/workflows/full-auto-development

# 应该看到 SKILL.md 和 WORKFLOW.md 文件
```

## 5分钟快速体验

### 场景：生成需求文档

**最简单的使用方式 - 只生成文档，不写代码**

```bash
# 1. 启动 Claude Code
claude

# 2. 输入命令
生成需求文档：实现用户登录功能

# 3. AI 会自动生成两个文件
# - REQUIREMENTS.md（详细需求文档）
# - TASK_LIST.md（任务清单）

# 4. 查看生成的文档
cat REQUIREMENTS.md
cat TASK_LIST.md
```

**你会得到：**
- ✅ 结构化的需求文档（11个章节）
- ✅ 详细的任务拆解（按优先级和依赖关系）
- ✅ 技术方案建议
- ✅ 风险识别

**用时：** 约 2-3 分钟

## 最佳实践

### 1. 写出好的需求描述

**✅ 好的需求描述：**
```bash
# 具体、清晰、包含关键细节
"实现用户登录注册功能，支持邮箱和手机号注册，包含找回密码、邮箱验证"

"添加商品评论系统，用户可以评分（1-5星）、评论、上传图片，管理员可以审核"

"实现文件上传功能，支持图片、PDF、文档，最大10MB，需要进度显示和断点续传"
```

**❌ 不够清晰的描述：**
```bash
# 太简单，缺少细节
"做个登录"
"加个评论功能"
"文件上传"
```

### 2. 合理使用确认环节

**应该确认的情况：**
- ✅ 功能模块划分是否合理
- ✅ 技术方案是否符合项目需求
- ✅ 任务拆解粒度是否合适
- ✅ 风险提示是否需要关注

**可以直接继续的情况：**
- ✅ 简单功能，需求明确
- ✅ 对生成的文档很满意
- ✅ 技术栈和项目一致

### 3. 充分利用进度记录

```bash
# 定期查看进度
"当前进度"

# 遇到问题查看日志
"查看错误日志"

# 长时间运行的任务可以中断休息
# 重连后继续即可
"继续"
```

### 4. 处理错误的策略

**自动修复失败时：**
1. 查看错误详情：`"查看错误详情"`
2. 查看完整日志：`"查看完整日志"`
3. 手动修复代码
4. 继续执行：`"已修复，继续"`

**或者从检查点重试：**
```bash
"查看检查点"
"从检查点2恢复"
```

### 5. 安全注意事项

**AI 会自动跳过这些文件：**
- `.env` 文件
- `credentials.json`
- `secrets.*`
- `*.key`

**大规模操作会二次确认：**
- 删除 10+ 个文件
- 修改核心配置文件
- 数据库迁移操作

### 6. 提高成功率的技巧

**技巧 1：分阶段实现**
```bash
# 不要一次性提出太复杂的需求
# 可以分阶段：

第一阶段：
"全自动开发：实现基础的用户登录功能"

第二阶段：
"全自动开发：添加找回密码功能"

第三阶段：
"全自动开发：添加第三方登录（Google、GitHub）"
```

**技巧 2：提供上下文**
```bash
# 如果项目已有特定技术栈或模式
"全自动开发：实现商品评论，使用项目现有的 Redux 状态管理和 Material-UI 组件库"
```

**技巧 3：明确约束条件**
```bash
"全自动开发：实现图片上传，要求支持拖拽上传、预览、压缩，单个文件最大5MB"
```

在项目根目录创建 `.claude-config.json`：

```json
{
  "full_auto_development": {
    "auto_commit": false,
    "max_retries": 3,
    "log_level": "info",
    "excluded_files": [".env", "secrets.json"],
    "custom_commands": {
      "test": "npm test",
      "lint": "eslint .",
      "build": "npm run build"
    }
  }
}
```

## 常见问题

### Q1: 中断后如何恢复？

```bash
# 重新连接后直接输入
"继续"

# AI 会自动检测未完成的任务并展示进度
```

### Q2: 生成的代码不满意怎么办？

```bash
# 方式 1：从检查点重新开始
"查看检查点"
"从检查点2恢复"

# 方式 2：手动修改后继续
# [手动修改代码]
"已修复，继续"

# 方式 3：取消重新开始
"取消"
# 然后重新发起，调整需求描述
```

### Q3: 可以只生成文档不写代码吗？

```bash
# 可以，使用需求分析命令
"生成需求文档：[功能描述]"
"需求分析：[功能描述]"

# 生成文档后不输入"继续开发"即可
```

### Q4: 支持哪些编程语言？

目前支持：
- JavaScript / TypeScript
- Python
- Go
- Rust
- Java
- Ruby

AI 会自动检测项目技术栈。

### Q5: 生成的测试覆盖率如何？

通常包括：
- 单元测试（核心功能）
- 集成测试（API 端点）
- 边界情况测试

覆盖率一般在 70-85%。
